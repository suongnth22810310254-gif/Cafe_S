💡 How the attack works:

  1. Attacker tạo trang với UI giả (nút xanh "Click Here to Claim")
  2. Nhúng victim page vào iframe với position: absolute
  3. Dùng CSS để iframe che khuất (opacity: 0, z-index thấp hơn UI giả)
  4. Điều chỉnh vị trí để nút "Delete Account" trùng với nút giả
  5. User click nút xanh → thực tế click vào nút đỏ trong iframe → hành động nguy hiểm xảy ra!

🛡️ How to prevent?

Victim page cần thêm header bảo vệ (như /clickjacking/protected): 

X-Frame-Options: DENY
Content-Security-Policy: frame-ancestors 'none'

Hoặc áp dụng toàn dự án bằng 1 file middleware securityHeaders.js!
